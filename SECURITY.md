# Security Policy

## Supported Versions

Only the latest release of each project receives security updates. Older versions
are supported on a best-effort basis.

## Reporting a Vulnerability

If you discover a security issue in any of my repositories, **please do not open
a public issue.** Instead, report it privately using one of these methods:

1. **GitHub Security Advisories** *(preferred)* —
   Go to the repository → **Security** → **Advisories** → **Report a vulnerability**.
   This keeps the report private and lets us coordinate a fix.
2. **Email** —
   Send details to **rudrasarker125@gmail.com** with the subject
   `[SECURITY] <repo-name>`.

Please include:

- The affected repository and commit/version
- Steps to reproduce or a proof-of-concept
- The potential impact

I aim to acknowledge reports within **48 hours** and will work with you on a
coordinated disclosure timeline.

## Scope

This policy applies to all repositories under the **[@rudra496](https://github.com/rudra496)**
account unless a repository ships its own `SECURITY.md`.
